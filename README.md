这是一个实时自动目标捕捉系统，应用于实时图像处理和目标检测领域。  
这个项目结合了深度学习、计算机视觉和系统优化技术，旨在实现高效和准确的目标识别和追踪。

## 核心思路

1.  **截图**: 使用 DXGI Desktop Duplication 实现低延迟画面捕获。
2.  **推理**: 使用 ONNX Runtime + TensorRT 后端运行端到端 YOLO 模型。
3.  **输入**: 调用 `IbInputSimulator` 库，通过硬件驱动模拟鼠标移动。


本项目能够实现，离不开以下优秀的开源项目：

[IbInputSimulator](https://github.com/Chaoses-Ib/IbInputSimulator)** by Chaoses-Ib: 用于实现驱动级的鼠标输入模拟。


## 总结

### 技术栈
**编程语言**：C++  
**深度学习框架**：ONNX Runtime  
**模型：YOLO 系列**（实时目标检测）  
**图像处理**：OpenCV  
**GPU 加速**：CUDA 和 cuDNN  
**平台**：Windows  

## 项目细节
1.**D3D 屏幕捕获**：  
使用 Direct3D 提供高速屏幕捕获功能。
实现资源管理和错误处理，确保系统稳定性。  

2.**模型加载与推理**：  
使用 ONNX Runtime 加载 YOLO 模型。  
设计了动态输入调整机制，提高模型兼容性。  
优化推理流程，提高推理效率。  

3.**后处理与目标检测**：  
实现 NMS（非极大值抑制）算法，减少误报。  
动态设定检测阈值，提高目标识别的准确性。  

4.**系统优化**：  
实现多线程支持，提高捕获与推理效率。  
灵活调整推理设备（CPU/GPU），满足不同硬件需求。  
优化内存管理，减少资源占用。  

## 取得成果
RTX4060 在 2560x1440 分辨率下，实现了 75 FPS 的目标检测速度。  
提升系统稳定性，通过错误处理机制减少崩溃率。  
支持多种硬件配置，实现跨平台兼容。  

## 遇到的挑战和解决方案
**DLL 依赖问题**：遇到 TensorRT 加载失败的问题，通过禁用不必要的 GPU 提供者解决。  
**资源管理**：在屏幕捕获过程中，针对资源失效进行自动恢复处理。  
**检测准确性**：通过调整 YOLO 模型参数和后处理算法，显著提高目标检测的精度。    

## 学到的经验  
深入了解深度学习模型优化与部署流程。  
提升了调试技能，尤其是在复杂环境下的错误排查。  
提高了代码的可读性与维护性，为后续项目开发积累经验。
